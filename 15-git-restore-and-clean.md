# 📝 التراجع عن الإضافة وحذف الملفات الزائدة (Restore & Clean) - الدرس الـ 15

في الدرس ده، اتعلمنا إزاي نتراجع عن خطوة الإضافة للمسرح (`Staging`) لو ضفنا ملف بالغلط، وإزاي نخلي `Git` ينظف لنا المشروع من الملفات الجديدة اللي عملناها للتجربة ومش محتاجينها.

## 1. التراجع عن إضافة ملف للمسرح (Unstaging / Restore)
لو عملت ملف جديد أو عدلت ملف، وكتبت أمر `git add`، بس بعدين اكتشفت إنك مش عايز ترفعه في الـ `Commit` ده. إنت هنا مش عايز تحذف الملف، إنت بس عايز "تنزله من على المسرح" وترجعه لحالة الـ `Untracked`.

**القاعدة الأساسية (Syntax):**

    git restore --staged <file_name>

**المثال العملي (Example):**

    git restore --staged new.txt

*(للتراجع عن إضافة كل الملفات مرة واحدة، بنستخدم النجمة كالتالي: `git restore --staged *`)*

> **💡 ملاحظة لمستخدمي VS Code:** > الخطوة دي هي بالظبط اللي بتحصل لما تفتح قائمة الـ Source Control في البرنامج وتدوس على علامة الناقص (`-`) أو السهم المايل جنب الملف عشان تعمله `Unstage`.

## 2. تنظيف المشروع من الملفات غير المتتبعة (Clean)
لو عملت ملفات كتير للتجربة (وهي لسه `Untracked` يعني معملتلهاش `add`)، وعايز تحذفهم كلهم بضربة واحدة بدل ما تروح تحذفهم من الويندوز بنفسك.

### أ. الرؤية قبل الحذف (Dry Run)
مهم جداً قبل ما تدي أمر الحذف، تخلي `Git` يعرضلك "إيه الملفات اللي هتتحذف؟" عشان تتأكد إن مفيش ملف مهم هيطير في الرجلين. بنستخدم حرف `-n`.

**القاعدة الأساسية والمثال العملي:**

    git clean -n

*(سيظهر لك قائمة بالملفات التي سيتم حذفها، مثل: `Would remove new.txt`)*

### ب. الحذف الفعلي (Force Clean)
بعد ما تتأكد من القائمة، بتستخدم حرف `-f` (اختصار Force) عشان تنفذ الحذف النهائي.

**القاعدة الأساسية والمثال العملي:**

    git clean -f

## 3. سيناريو عملي متقدم (الدمج بين الأوامر)
تخيل إنك عملت 3 ملفات جديدة للتجربة (`new1.txt`, `new2.txt`, `new3.txt`).
اكتشفت إنك محتاج تحتفظ بـ `new2.txt` فقط، وعايز تحذف الباقي. تعمل إيه؟

1. **أولاً:** احمي الملف اللي إنت عايزه بإضافته للمسرح:
    `git add new2.txt`
2. **ثانياً:** اعرض الملفات المتبقية اللي هتتحذف للتأكيد:
    `git clean -n`
3. **ثالثاً:** نفذ الحذف النهائي للملفات غير المتتبعة:
    `git clean -f`

*(بكده `Git` هيحذف `new1` و `new3`، وهيسيب `new2` بأمان لأنه بقى متتبع جوه الـ Staging Area).*
