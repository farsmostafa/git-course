# 📝 احتراف أوامر الـ Stash المتقدمة (الدرس الرابع عشر)

في الدرس ده، كملنا رحلتنا مع الـ `Stash` وتعلمنا إزاي نتعامل مع أكتر من "صندوق مخفي" في نفس الوقت، وإزاي نسميهم، ونختار واحد معين نرجعه أو نحذفه، وإزاي ننظف الدنيا تماماً.

## 1. حفظ التعديلات برسالة مخصصة (Save with Message)
لما بتعمل `Stash` كتير، الـ `Git` بيكتب رسائل تلقائية ممكن تلخبطك. عشان تنظم الدنيا وتعرف كل صندوق فيه إيه، لازم تسميه بنفسك.

**القاعدة الأساسية (Syntax):**

    git stash save "Your descriptive message here"

**المثال العملي (Example):**

    git stash save "fixing header style issues"

---

## 2. استرجاع التعديلات (Pop مقابل Apply)
هنا فيه فرق جوهري لازم تفهمه عشان تحافظ على شغلك:

### أ. أمر الـ `Pop`:
بياخد التعديلات من الصندوق يرجعها لكودك، **وبيمسح** الصندوق ده من القائمة نهائياً.
**المثال العملي:**
    git stash pop

### ب. أمر الـ `Apply`:
بياخد التعديلات يرجعها لكودك، بس **بيسيب** النسخة في الصندوق زي ما هي (مش بيمسحها). ده مفيد لو عايز تجرب التعديلات دي في كذا فرع (`Branch`) مختلف.
**المثال العملي:**
    git stash apply

---

## 3. التعامل مع ستاش معين (Targeting specific stash)
لما بتكتب `git stash list` بيظهرلك كذا ستاش مترقمين بـ `Index` بيبدأ من 0 (مثلاً: `stash@{0}`, `stash@{1}`). لو عايز تطبق أمر على واحد معين فيهم:

**القاعدة الأساسية (Syntax):**

    git stash <command> stash@{index_number}

**المثال العملي (Example):** (لاسترجاع الصندوق رقم 1 وحذفه)

    git stash pop stash@{1}

---

## 4. معاينة محتويات الصندوق (Show)
لو عندك ستاش ومش فاكر إيه الملفات اللي كانت جواه، بدل ما ترجعه وتكتشف إنه مش هو، تقدر "تلمح" اللي جواه الأول.

**القاعدة الأساسية (Syntax):**

    git stash show stash@{index_number}

**المثال العملي (Example):**

    git stash show stash@{0}

---

## 5. حذف التعديلات المخفية (Drop & Clear)

### أ. حذف ستاش واحد معين (`Drop`):
لو في صندوق مبقتش محتاجه وعايز ترميه هو بس.
**القاعدة الأساسية والمثال العملي:**

    git stash drop stash@{2}

### ب. مسح كل الـ `Stashes` تماماً (`Clear`):
لو عايز تنظف كل الصناديق المخفية اللي عندك بضربة واحدة وتولع فيهم بجاز (حذر: ده بيمسح كل الداتا اللي متخزنة فيهم ومبتعرفش ترجعها تاني).
**القاعدة الأساسية والمثال العملي:**

    git stash clear

---

## 📌 ملخص سريع للعمل الجماعي (Team Workflow):
زي ما شفنا في الفيديو، دايماً قبل ما ترفع شغلك بعد ما ترجعه من الـ `Stash`:
1. اعمل `git pull` عشان تسحب أي جديد من السيرفر.
2. حل أي تعارضات لو ظهرت.
3. اعمل الـ `Commit` والـ `Push` بتوعك بأمان.

**💡 تمرين ليك:** جرب تعمل كذا ستاش، وسميهم بأسماء مختلفة، واستخدم `git stash list` عشان تشوفهم، وبعدين جرب تمسح واحد بـ `drop` وتمسح الباقي بـ `clear`.
