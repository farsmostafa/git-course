# 📝 إخفاء التعديلات مؤقتاً (Git Stash) - الدرس الثالث عشر

في الدرس ده، اتعلمنا إزاي نتعامل مع موقف بيحصل كتير جداً في الشركات: إنك تكون شغال في ميزة معينة ولسه مخلصتهاش، وفجأة يُطلب منك تعديل عاجل في ملف تاني. إزاي تخفي شغلك الحالي مؤقتاً (بدون ما تحذفه أو تعمل له `Commit` وهو ناقص)، عشان ترجعله بعدين؟ الحل هو أمر `Stash`.

## 1. ما هو الـ `Stash`؟

- **المفهوم:** هو مساحة تخزين مؤقتة (زي درج أو صندوق مخفي) في `Git`.
- **الوظيفة:** بياخد كل الملفات اللي إنت شغال عليها حالياً (المتعدلة أو الجديدة اللي اتعملها `Add` للمسرح) ويخبيها في الصندوق ده، ويرجعلك مساحة العمل (`Working Directory`) نظيفة تماماً كأنك معملتش أي حاجة.

## 2. كيفية إخفاء التعديلات (Stashing)

بعد ما تعمل تعديلاتك (سواء بإنشاء ملفات جديدة أو تعديل القديمة)، ضفها أولاً لمنطقة التجهيز (`git add`)، ثم قم بإخفائها.

**القاعدة الأساسية والمثال العملي:**

    git stash

- **ماذا يحدث بعد الأمر؟**
  - سيتم سحب الملفات وحفظها.
  - لو كتبت `git status`، هتلاقي النتيجة `working tree clean` (مفيش أي تعديلات).
  - لو دورت على الملفات في مجلد المشروع على جهازك، مش هتلاقيها! (لأنها اتخبت جوه الصندوق المخفي).

## 3. عرض محتويات الصندوق (List)

عشان تتأكد إن ملفاتك متخزنة بأمان ومضاعتش، تقدر تعرض قائمة بكل التخزينات (`Stashes`) اللي إنت عملتها.

**القاعدة الأساسية والمثال العملي:**

    git stash list

- هيظهر لك قائمة مرقمة، تبدأ من `stash@{0}` (الرقم 0 هو دائماً أحدث شيء تم إخفاؤه).

## 4. استرجاع التعديلات من الصندوق (Pop)

بعد ما تخلص الشغل الطارئ اللي جالك وترفعه على `GitHub`، هتحتاج ترجع ملفاتك اللي كنت مخبيها عشان تكمل عليها.

**القاعدة الأساسية (Syntax):**

    git stash pop

- **ماذا يعني `pop`؟**
  - بياخد الملفات من الصندوق يرجعها لمجلد العمل بتاعك عشان تشوفها وتكمل شغل عليها.
  - في نفس الوقت، **بيحذف الصندوق ده تماماً** (بيرمي المحتوى من الذاكرة المؤقتة لأنه خلاص مبقاش محتاجه).
  - لو كتبت `git stash list` بعد الـ `pop`، هتلاقي الصندوق اختفى.

## 5. مراجعة مهمة (سحب التحديثات قبل الرفع)

في الفيديو، ظهرت رسالة خطأ مهمة جداً أثناء محاولة الرفع (`Push`):
`failed to push some refs... Updates were rejected because the remote contains work that you do not have locally.`

- **السبب:** يعني إن في تعديلات موجودة على `GitHub` (أو السيرفر) مش موجودة على جهازك حالياً.
- **الحل الدائم:** لازم تسحب التعديلات دي الأول لجهازك عشان تدمجها، وبعدين ترفع كودك.

**الأوامر بالترتيب الصحيح:**

    git pull origin main
    # بعد التأكد من عدم وجود تعارض
    git push origin main
