# 📝 إنشاء مستودع محلي وربطه بمستودع سحابي (الدرس التاسع)

في الدرس ده، اتعلمنا إزاي نحول أي مجلد عادي على جهازنا لمستودع `Git` رسمي، وإزاي نرفعه على `GitHub` لو المجلد ده فيه مشروع شغالين عليه فعلاً ومش واخدينه `Clone`.

## 1. تحويل المجلد لمستودع (`git init`)

لما يكون عندك فولدر وعايز تبدأ تستخدم فيه `Git` عشان يراقب التغييرات، لازم تعمل له عملية "تهيئة" أو `Initialization`.

* **ماذا يحدث عند كتابة الأمر؟** بيتم إنشاء فولدر مخفي اسمه `.git` جوه مشروعك، الفولدر ده هو اللي فيه كل "السحر" وتاريخ التعديلات.

**القاعدة الأساسية (Syntax):**

    git init

**المثال العملي (Example):** (بداخل مجلد مشروعك)

    git init

## 2. ربط المستودع المحلي بالسحابي (`git remote add`)

بما إننا بدأنا المشروع من جهازنا، فـ `Git` لسه ميعرفش هو هيترفع فين على الإنترنت. لازم إحنا نعرّفه الرابط بتاع `GitHub`.

* **ملاحظة هامة:** بما إننا فعلنا الـ `SSH Key` في الدرس اللي فات، لازم نختار رابط الـ `SSH` من موقع `GitHub` (اللي بيبدأ بـ `git@github.com`) مش رابط الـ `HTTPS`.

**القاعدة الأساسية (Syntax):**

    git remote add <remote_name> <ssh_url>

**المثال العملي (Example):**

    git remote add origin git@github.com:farsmostafa/git-course.git

## 3. الرفع لأول مرة مع الربط التلقائي (`-u`)

في أول مرة ترفع فيها مشروع بدأته من جهازك، بنزود حرف `-u` (اختصار لـ `upstream`). ده بيخلي `Git` يربط الفرع المحلي بالفرع اللي على السيرفر بشكل دائم، فعشان بعد كده لما تحب ترفع أو تسحب، تكتب `git push` أو `git pull` بس وهو هيفهم لوحده.

**القاعدة الأساسية (Syntax):**

    git push -u <remote_name> <branch_name>

**المثال العملي (Example):**

    git push -u origin main

## 4. علاقة الـ `Config` بـ `GitHub`

* في الدرس ده ظهرت أهمية درس الإعدادات (`Config`)؛ الإيميل اللي إنت سجلته في إعدادات الجيت هو اللي بيخلي صورتك واسمك يظهروا جنب الـ `Commit` على موقع `GitHub`. لو سجلت إيميل غلط، الكوميت هيظهر باسم شخص تاني أو بدون صورة بروفايلك.

## 5. نصيحة العمل في تيم (البول قبل البوش)

* دايماً قبل ما ترفع شغلك (`Push`)، لازم تسحب أحدث حاجة من السيرفر (`Pull`)؛ لأن ممكن زميلك يكون رفع تعديلات في نفس الوقت، ولو رفعت مباشرة هيحصل تعارض (`Conflict`).
* الصح: اسحب آخر نسخة، اتأكد إن كودك متوافق معاها، وبعدين ارفع تعديلاتك.
