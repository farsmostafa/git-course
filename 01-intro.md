# 📝 مقدمة في Git و GitHub (الدرس الأول)

## 1. ما هو Git؟
* هو نظام لإدارة النسخ الموزعة (**Distributed Version Control System**).
* **فكرة الـ Versioning (النسخ):** زي ما أي برنامج (مثلاً Photoshop) بينزله إصدارات وتحديثات، الكود بتاعك بيمر بمراحل وتحديثات (إضافة ميزة، حل مشكلة، تحسين كود). Git بيسجل وبيتحكم في كل النسخ دي.
* **مجاني ومفتوح المصدر (Free & Open Source):** أداة مجانية بالكامل وتقدر تساهم في تطويرها.

## 2. ما هو GitHub؟ وهل هو Git؟
* **GitHub** هو مجرد **موقع ويب** لاستضافة المشاريع والأكواد، بيسهل استخدام Git وبيوفر أدوات إضافية زي تتبع المشاكل (Bug/Issue Tracker).
* **Git $\neq$ GitHub:** تقدر تستخدم Git بالكامل بدون GitHub! ممكن ترفع شغلك على سيرفر الشركة الداخلي، أو تستخدم مواقع بديلة بتقدم نفس الخدمة زي GitLab و Bitbucket.

## 3. ليه لازم نتعلم Git؟ (المشاكل اللي بيحلها)
قبل Git، كانت بيئة العمل عبارة عن فوضى (انتظار زميلك يقفل ملف عشان تدخل تعدل عليه، فقدان التعديلات، إرسال الأكواد بالإيميل وحدوث تضارب). Git حل كل ده من خلال:
* **العمل الجماعي المتزامن:** الفريق كله بيشارك في نفس المشروع وكل التعديلات بتندمج مع بعض بسلاسة.
* **التراجع عن الأخطاء (Reverting):** لو حصلت مشكلة، تقدر تعرف مين كتب السطر ده، وإمتى، وتقدر تلغي التعديل ده تحديداً وترجع للنسخة السليمة بدون ما تأثر على باقي الكود.
* **تقسيم المهام بسهولة:** تقدر تكتب أنت كود الـ HTML، وزميلك يكمل عليه بـ CSS و JavaScript، وكل ده بيتجمع في ميزة (Feature) واحدة شغالة.
* **حل التعارض (Resolving Conflicts):** لو اتنين مطورين عدلوا في نفس الـ Function، Git بيقارن النسختين ويسمحلك تختار الأكواد السليمة عشان تدمجها وتصلح التعارض.
* **التنظيم وتتبع التعديلات:** كل تعديل بيترفع بيكون معاه رسالة توضح بيعمل إيه بالظبط، وكل حاجة متسجلة في Log فايل.
  > **💡 مثال على التنظيم الاحترافي:** بدلاً من التعديلات العشوائية، الـ History بيكون واضح ومفهوم للجميع:
  > `feat(auth): validate email and password on signup`
  > `fix(profile): handle null avatar gracefully`

## 4. سطر الأوامر (CLI) ولا الواجهة الرسومية (GUI)؟
* Git له واجهات رسومية (زي برنامج GitHub Desktop) بتقدر تستخدمها بالماوس.
* **نصيحة ذهبية:** احتراف الـ **Command Line (CLI)** أو سطر الأوامر هو الأفضل بلا منازع، لأنه بيطور مستواك جداً كمبرمج وبيفتحلك باب الدخول لعالم أنظمة الـ Unix/Linux والشغل المتقدم.
