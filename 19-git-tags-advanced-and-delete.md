# 📝 الإصدارات المتقدمة والحذف (Advanced Tags & Releases) - الدرس الـ 19

في الدرس ده، قفلنا موضوع الـ `Tags` وتعلمنا إزاي نبحث عن إصدارات معينة، وإزاي نحول الـ `Tag` لـ `Release` حقيقي بنوتس وروابط تحميل، وأهم حاجة: إزاي نحذفهم لو غلطنا.

## 1. البحث عن الإصدارات (Filtering Tags)

لو مشروعك كبير وفيه عشرات الإصدارات، مش منطقي تدور بعينك. تقدر تستخدم أمر البحث مع علامة النجمة (`*`) كجوكر للبحث عن نمط معين.

**القاعدة الأساسية (Syntax):**

    git tag -l "<pattern>*"

**المثال العملي (Example):** (للبحث عن كل إصدارات الجيل الأول)

    git tag -l "v1.*"

---

## 2. الفرق بين الـ `Tag` والـ `Release` على `GitHub`

- **الـ `Tag`:** هو مجرد علامة تقنية جوه الـ `Git`.
- **الـ `Release`:** هو واجهة عرض جميلة على `GitHub` مبنية على الـ `Tag` ده. تقدر تضيف فيها:
  - **Release Notes:** شرح إيه الجديد اللي حصل في النسخة دي.
  - **Binaries:** ملفات البرنامج جاهزة للتشغيل (زي ملفات الـ `.exe` أو الـ `APK`).
  - **Pre-release:** علامة توضح إن النسخة دي تجريبية (بيتا) وليست مستقرة تماماً.
  - **Draft:** مسودة تقدر تحفظها وتعدل عليها لحد ما تقرر تنشر الإصدار للعامة.

---

## 3. مقارنة الإصدارات (Comparison)

ميزة قوية في `GitHub` بتسمحلك تعمل `Compare` بين أي إصدارين، عشان تشوف بالظبط إيه ملفات الكود اللي اتغيرت أو انضافت بين الإصدار القديم والجديد.

---

## 4. حذف العلامات (Deleting Tags)

لو عملت `Tag` غلط أو باسم مش مظبوط، لازم تحذفه من مكانين: جهازك (`Local`) والسيرفر (`Remote`).

### أ. الحذف من الجهاز المحلي:

**القاعدة الأساسية (Syntax):**

    git tag -d <tag_name>

**المثال العملي (Example):**

    git tag -d v1.0

### ب. الحذف من `GitHub` (Remote):

حذفها من جهازك مش بيمسحها من الموقع، لازم تبعت أمر دفع خاص بالحذف للسيرفر.

**القاعدة الأساسية (Syntax):**

    git push <remote_name> --delete <tag_name>

**المثال العملي (Example):**

    git push origin --delete v1.0

---

## 💡 ملاحظة أخيرة:

دايماً لما تحذف `Tag` من الموقع يدوياً (عن طريق الجي يو آي - GUI)، اتأكد إنك تحذفه من جهازك بالكوماند لاين عشان لما تيجي ترفع تاني ميحصلش تعارض أو يرجع يترفع القديم اللي إنت مسحته.
