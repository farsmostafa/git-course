# 📝 دورة حياة الملفات في Git وأوامر الإضافة والاعتماد (الدرس الرابع)

في الدرس ده، اتعلمنا إزاي نشتغل على المشروع فعلياً وننقل الملفات من مجرد ملفات عادية على جهازنا لحد ما تبقى محفوظة في الـ Local Repository.

## 1. مناطق العمل في Git (دورة حياة الملف)
عشان تفهم Git بيشتغل إزاي، لازم تتخيل إن عندك 3 مناطق أساسية على جهازك (Local)، ومنطقة رابعة على الإنترنت (Remote):

1. **Working Directory (مجلد العمل):** ده الفولدر العادي اللي بتكتب فيه كودك وتعمل فيه الملفات الجديدة.
2. **Staging Area (منطقة التجهيز):** دي زي خشبة المسرح. انت بتختار ملفات معينة تحطها هنا عشان تجهزها للحفظ.
3. **Local Repository (المستودع المحلي):** ده المخزن اللي على جهازك، لما بتعمل تأكيد (Commit)، الملفات بتتنقل من المسرح للمخزن ده، وبكده تكون حفظت نسخة من شغلك.
4. **Remote Repository (المستودع السحابي):** ده زي GitHub، بترفع عليه الكود من المخزن المحلي بتاعك عشان باقي الفريق يشوفه.

## 2. أهم الأوامر اللي اتعلمناها

### أ. معرفة الحالة (Status)
الأمر ده هو صديقك الصدوق، بتكتبه عشان تعرف انت فين وإيه الملفات اللي اتعدلت أو اتضافت.

**القاعدة الأساسية والمثال العملي:**

    git status

### ب. إضافة الملفات لمنطقة التجهيز (Add)
لما تعمل ملف جديد أو تعدل في ملف (زي `index.html`)، Git مش بيحس بيه أوتوماتيك. لازم تقوله "ضيف الملف ده للمسرح".

**1. إضافة ملف واحد:**
**القاعدة الأساسية (Syntax):**

    git add <file_name>

**المثال العملي (Example):**

    git add index.html

**2. إضافة كل الملفات الجديدة والمعدلة مرة واحدة:**
**القاعدة الأساسية والمثال العملي:**

    git add *
    أو
    git add .

### ج. التراجع عن الإضافة (Unstage / Reset)
لو ضفت ملف بالغلط للـ Staging Area وعايز ترجعه تاني لمجلد العمل العادي (تنزل البطل من عالمسرح).

**القاعدة الأساسية (Syntax):**

    git reset HEAD <file_name>

**المثال العملي (Example):**

    git reset HEAD main.css

### د. حفظ التعديلات في المستودع المحلي (Commit)
دي أهم خطوة، وهي حفظ اللقطة (Snapshot) من الملفات اللي في المسرح للـ Local Repo بتاعك. لازم تكتب رسالة توضح انت عملت إيه.

**القاعدة الأساسية (Syntax):**

    git commit -m "<commit_message>"

**المثال العملي (Example):**

    git commit -m "feat: setup main project structure"

## 3. ملاحظات مهمة
* **هل ممكن أعمل أكتر من Commit؟** أيوة طبعاً! لو خلصت جزء وعملت Commit، واكتشفت إنك نسيت ملف (مثلاً نسيت تضيف `main.css`)، تقدر تضيف الملف لوحده وتعمل Commit تاني عادي جداً. كلهم بيتجمعوا في اللوكال عندك.
* **هل شغلي كده اترفع على GitHub؟** لأ! كل الأوامر اللي فاتت دي بتتم على جهازك أنت بس. عشان ترفعهم على الإنترنت لازم تستخدم أمر الدفع (`git push`) وده اللي هنعرفه بعدين.
