# 📝 الفروع في جيت (Git Branching) - الدرس الثاني عشر

في الدرس ده، اتعلمنا إزاي نشتغل على ميزات جديدة أو نصلح أخطاء في مسار معزول تماماً عن الكود الأساسي للمشروع باستخدام "الفروع" (`Branches`)، وإزاي ندمج الشغل ده بعد ما نخلصه.

## 1. ما هو الفرع (`Branch`)؟

الفرع هو عبارة عن مسار تطوير مستقل. تخيل إن الكود الأساسي بتاعك ماشي في خط مستقيم (فرع `main`)، وإنت اتطلب منك تضيف ميزة جديدة. بدل ما تعدل على الـ `main` مباشرة، بتاخد منه "نسخة/فرع" تشتغل عليه لوحدك، ولما تخلص وتتأكد إن كودك سليم، بترجعه وتدمجه تاني مع الـ `main`.

## 2. الأوامر الأساسية للتعامل مع الفروع

### أ. عرض الفروع المتاحة

لعرض كل الفروع الموجودة في مشروعك، ومعرفة الفرع الذي تقف عليه حالياً (ستجد بجانبه علامة النجمة `*`).

**القاعدة الأساسية والمثال العملي:**

    git branch

### ب. إنشاء فرع جديد

لإنشاء فرع جديد (نسخة من المكان اللي إنت واقف فيه) بدون الانتقال إليه.

**القاعدة الأساسية (Syntax):**

    git branch <branch_name>

**المثال العملي (Example):**

    git branch scroll

### ج. الانتقال بين الفروع

للذهاب إلى فرع آخر موجود بالفعل لتبدأ العمل بداخله.

**القاعدة الأساسية (Syntax):**

    git checkout <branch_name>

**المثال العملي (Example):**

    git checkout scroll

### د. إنشاء فرع جديد والانتقال إليه فوراً (أمر مختصر)

بدل ما تكتب أمرين للإنشاء والانتقال، الأمر ده بيعمل الخطوتين مع بعض في ضربة واحدة (و ده الأكثر استخداماً).

**القاعدة الأساسية (Syntax):**

    git checkout -b <branch_name>

**المثال العملي (Example):**

    git checkout -b dev-feature

### هـ. إعادة تسمية الفرع الحالي

لو أنشأت فرع واسمه مش عاجبك، تقدر تغير اسمه (تأكد إنك واقف جواه أولاً).

**القاعدة الأساسية (Syntax):**

    git branch -m <new_name>

**المثال العملي (Example):**

    git branch -m development

## 3. حذف الفروع (Delete)

بعد ما تخلص شغلك في الفرع وتدمجه، يفضل تحذفه عشان تحافظ على نظافة المشروع.

### أ. الحذف الآمن (Safe Delete - `d` صغيرة)

هذا الأمر بيحذف الفرع، بس كنوع من الحماية، `Git` هيرفض يحذفه لو الفرع ده فيه كود إنت لسه مدمجتوش (عشان مجهودك ميضيعش).

**القاعدة الأساسية (Syntax):**

    git branch -d <branch_name>

**المثال العملي (Example):**

    git branch -d scroll

### ب. الحذف الإجباري (Force Delete - `D` كبيرة)

هذا الأمر بيجبر `Git` على حذف الفرع فوراً، حتى لو كان جواه تعديلات لسه لم يتم دمجها (استخدمه بحذر لو إنت متأكد إنك مش عايز الكود اللي جواه).

**القاعدة الأساسية (Syntax):**

    git branch -D <branch_name>

**المثال العملي (Example):**

    git branch -D scroll

## 4. دمج الفروع (Merging)

بعد ما بتخلص شغلك في الفرع الفرعي (زي `development`)، بتحتاج تدمجه مع الفرع الأساسي (`main`). الدمج بيتم على خطوتين:

1. لازم ترجع تقف على الفرع الأساسي اللي هيستقبل الكود (`git checkout main`).
2. تسحب الكود من الفرع الفرعي تدمجه في الأساسي.

**القاعدة الأساسية (Syntax):**

    git merge <branch_name>

**المثال العملي (Example):** (بافتراض أنك تقف الآن على فرع `main`)

    git merge development

_(بعد تنفيذ هذا الأمر، يمكنك رفع فرع `main` للإنترنت باستخدام `git push origin main`، ثم حذف فرع `development` لأنه أدى مهمته)._

## 5. رفع الفرع للإنترنت لعمل طلب دمج (`Pull Request`)

في بيئة العمل الحقيقية (الشركات)، إحنا مش بندمج الكود محلياً زي الخطوة اللي فاتت. إحنا بنرفع "الفرع الفرعي" نفسه على السيرفر، ونطلب من التيم يراجعه الأول.

**القاعدة الأساسية (Syntax):**

    git push <remote_name> <branch_name>

**المثال العملي (Example):** (لرفع فرع scroll إلى GitHub)

    git push origin scroll

**ماذا يحدث بعد ذلك؟**

- بمجرد رفع الفرع، ستظهر لك رسالة في الـ `Terminal` تحتوي على رابط مباشر لإنشاء `Pull Request`.
- عند فتح `GitHub`، ستجد زراً أخضر يقول `Compare & pull request`.
- تضغط عليه، وتكتب وصفاً للتعديل، ثم يراجعه قائد الفريق ويقوم بدمجه (`Merge`) من الموقع مباشرة.
- بعد دمج الطلب على الموقع، يمكنك الضغط على `Delete branch` من `GitHub` لحذفه السحابي للحفاظ على نظافة المستودع.
