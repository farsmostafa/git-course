# 📝 إنشاء حساب GitHub واستنساخ أول مستودع (الدرس الثالث)

في هذا الدرس، بدأنا التطبيق العملي بإنشاء حساب على `GitHub`، وتجهيز أول مستودع (`Repository`)، ثم سحبه إلى جهازك المحلي (`Local Machine`).

## 1. هل `GitHub` هو الخيار الوحيد؟

- فهمك لمفاهيم وأوامر `Git` هو الأهم. إذا فهمت الأساسيات، يمكنك تطبيق نفس الأوامر تماماً على أي منصة أخرى مثل `GitLab` أو `BitBucket` أو حتى خادم الشركة الداخلي (`Centralized Server`). الأوامر ثابتة لا تتغير.

## 2. إنشاء حساب على `GitHub`

- التسجيل يتم بخطوات بسيطة (اسم المستخدم، البريد، كلمة المرور).
- الخطة المجانية (`Free Tier`) كافية جداً لك كمطور، وتوفر كل ما تحتاجه لرفع مشاريعك. (الخطة المدفوعة تفيد الشركات والفرق الكبيرة للحصول على ميزات إضافية).

## 3. إنشاء مستودع جديد (`New Repository`)

عند إنشاء `Repo` جديد على الموقع، ستقابلك بعض الخيارات المهمة:

- **اسم المستودع (`Repository Name`):** يجب أن يكون معبراً عن المشروع ولا يحتوي على مسافات (مثال: `git-course`).
- **الوصف (`Description`):** نبذة مختصرة عن المشروع (اختياري).
- **حالة المستودع (`Visibility`):**
  - `Public`: متاح للجميع لرؤيته (مفيد جداً لمعرض أعمالك).
  - `Private`: خاص بك وبفريقك فقط ولا يمكن لأحد رؤية الكود غيركم.
- **ملف `README.md`:** خطوة هامة جداً، وهو ملف نصي لتوثيق المشروع، كتابة الملاحظات، وخطوات التشغيل لكي يقرأها أي مطور يدخل إلى المستودع ويفهم المشروع.

## 4. ما هي لغة `Markdown` (`.md`)؟

- هي لغة تنسيق نصوص بسيطة جداً تُستخدم لكتابة ملفات الـ `README.md` وغيرها من ملفات التوثيق (`Documentation`).
- تقوم بتحويل الرموز البسيطة إلى تنسيق مرئي واضح.
- **أمثلة:**
  - استخدام `#` يمثل عنوان رئيسي كبير (`Heading 1`).
  - استخدام `##` يمثل عنوان فرعي أصغر (`Heading 2`).
- منصة `GitHub` توفر دليلاً كاملاً (`Syntax Guide`) لتعلم طريقة كتابة الأكواد، القوائم (`Lists`)، والروابط باستخدام هذه اللغة.

## 5. استنساخ المستودع (`Clone`)

هذه هي الخطوة العملية الأولى لربط المشروع الموجود على الخادم السحابي (`Remote Repo`) بجهازك.

### الخطوات العملية في سطر الأوامر (`Terminal` / `Command Line`):

1. من صفحة الـ `Repo` على موقع `GitHub`، اضغط على الزر الأخضر `Code` (أو `Clone or download`) وانسخ الرابط الذي ينتهي بـ `.git`.
2. افتح سطر الأوامر (`Terminal`).
3. استخدم أوامر نظام التشغيل العادية للذهاب للمجلد الذي تريد حفظ المشروع فيه:
   - أمر `cd` للدخول إلى مجلد معين (مثال: `cd Downloads`).
   - أمر `mkdir` لإنشاء مجلد جديد (مثال: `mkdir my-github-projects`).
4. الآن، اكتب أمر الاستنساخ متبوعاً بالرابط الذي قمت بنسخه:
   ```bash
   git clone [https://github.com/Username/git-course.git](https://github.com/Username/git-course.git)
   ```
5. بعد انتهاء التحميل، ادخل إلى مجلد المشروع الجديد الذي تم إنشاؤه:

```Bash
cd git-course
```

6. لضمان نجاح العملية، يمكنك كتابة أمر dir (في الويندوز) أو ls (في الماك/لينكس) لترى محتويات المجلد، وستجد بداخله ملف README.md الذي أنشأته على الموقع.

#### 💡 ملاحظة هامة: لا نستخدم الواجهة الرسومية لموقع GitHub لرفع أو تعديل الملفات (إلا في حالات نادرة جداً مثل تعديل سريع في ملف نصي). كل العمل الاحترافي سيتم من خلال سطر الأوامر (Terminal) على جهازك.
