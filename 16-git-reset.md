# 📝 التراجع عن التعديلات وحذفها نهائياً (Git Reset) - الدرس الـ 16

في الدرس ده، هنتعلم إزاي نتعامل مع "الكوارث". لو عملنا `Commit` بالغلط، أو كود مليان مشاكل اترفع وعايزين نحذفه تماماً ونرجع لنسخة قديمة كانت شغالة وزي الفل.

## 1. مفهوم الـ `HEAD` (المؤشر)

عشان تفهم الـ `Reset` بيشتغل إزاي، لازم تفهم إن `Git` مش بيحذف الكوميتات زي ما بتحذف فولدر في الويندوز.

- **الـ `HEAD`:** هو ببساطة "السهم" أو المؤشر اللي بيشاور على آخر `Commit` إنت واقف عليه حالياً.
- **فكرة الحذف:** إحنا بنمسك السهم ده (`HEAD`) ونرجعه لورا نخليه يشاور على `Commit` قديم (كان سليم). تلقائياً، أي `Commit` جه بعده بيتم إلغاؤه وحذفه.

## 2. خطوات الحذف والرجوع بالزمن (Reset Hard)

عشان نلغي التعديلات البايظة، بنمشي على 3 خطوات بالترتيب:

### الخطوة الأولى: معرفة الـ `Hash` الخاص بالكوميت السليم

لازم الأول نفتح سجل التاريخ عشان نجيب الكود السري (`Hash`) بتاع الـ `Commit` السليم اللي عايزين نرجعله.

**القاعدة الأساسية والمثال العملي:**

    git log

_(سيظهر لك قائمة بالكوميتات. انسخ أول أرقام وحروف من الـ `Hash` الخاص بالكوميت الجيد. للخروج من هذه الشاشة في الـ `Terminal`، اضغط على حرف `q`)._

### الخطوة الثانية: تنفيذ أمر التراجع الصارم (`--hard`)

استخدام `--hard` معناه: "رجع المؤشر للكوميت ده، وامسح أي تعديلات أو ملفات اتعملت بعده تماماً من على جهازي".

**القاعدة الأساسية (Syntax):**

    git reset --hard <commit_hash>

**المثال العملي (Example):**

    git reset --hard 7a2b9c4

_(بعد تنفيذ الأمر، ستظهر رسالة `HEAD is now at...` وهذا يعني أن المؤشر رجع بنجاح للنسخة السليمة)._

### الخطوة الثالثة: الإجبار على الرفع (`Force Push`)

لو حاولت ترفع الكود دلوقتي بـ `git push` العادي، `GitHub` هيرفض! ليه؟ لأن الموقع عليه كوميتات أكتر من اللي على جهازك (لأنك لسه ماسحهم من عندك). فلازم نستخدم القوة الجبرية عشان نجبر السيرفر يمسحهم هو كمان ويبقى زي جهازك.

**القاعدة الأساسية (Syntax):**

    git push <remote_name> <branch_name> --force

**المثال العملي (Example):**

    git push origin main --force
    أو باختصار:
    git push origin main -f

---

## ⚠️ تحذير هام جداً (Red Alert):

أمر `git reset --hard` خطير جداً، لأنه بيمسح التعديلات بشكل نهائي ولا يمكن التراجع عنها. تأكد مليون في المية إنك مش محتاج الأكواد اللي هتتحذف قبل ما تدوس `Enter`.

## 💡 نصيحة للعمل في فرق (Teamwork):

لو إنت شغال في تيم كبير، أمر `Push --force` ممنوع تماماً على الـ `main branch`، لأنه هيمسح شغل زمايلك لو كانوا رفعوا حاجة بعدك. بنستخدمه بس لو إحنا شغالين على فرعنا الخاص، أو لو إحنا الـ Admins وبنصلح كارثة.
